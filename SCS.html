1 通信协议概要

通讯电平采用兼容高速通信的TTL电平方式与具有较强抗干扰能力的RS485方式，通讯仍是采用异步双工，发送与接收信号是异步处理。

控制器和舵机之间采用问答方式通信，控制器发出指令帧，舵机返回应答帧。

一个总线控制网络中允许有多个舵机，所以每个舵机都分配有一个网络内唯一的ID号。控制器发出的控制指令中包含ID信息，只有匹配上ID号的舵机才能完整接收这条指令，并返回应答信息。

通信方式为串行异步方式，一帧数据分为1位起始位，8位数据位和1位停止位，无奇偶校验位，共10位。

内存表的部分参数采用两个字节的值域时，两个字节顺序根据舵机型号区分，电位器类型舵机为大端格式（高字节在前低字节在后），磁编码类型舵机为小端格式（低字节在前高字节在后）。每款舵机因功能略有不同，因此实际控制时请参照具体型号的内存表为准。

2 指令帧

字头 ID号 数据长度 指令 参数 校验和
0xFF 0xFF ID Length Instruction Parameter1 …Parameter N Check Sum
  • 字头：连续收到两个0xFF，表示有数据包到达。
  • ID号：0～253（0x00～0xFD）；广播ID：254（0xFE）
  • 数据长度：参数长度N + 2
  • 校验和：Check Sum = ~(ID + Length + Instruction + 参数…)，取低字节

3 应答帧

字头 ID号 数据长度 状态码 参数 校验和
0xFF 0xFF ID Length ERROR Parameter1 …Parameter N Check Sum

ERROR=0 表示无错误。

4.1 PING 查询状态

示例：ID=1

指令帧：FF FF 01 02 01 FB
应答帧：FF FF 01 02 00 FC

4.2 READ DATA 读数据

示例：读ID1当前位置（地址0x38，长度2）

指令帧：FF FF 01 04 02 38 02 BE
应答帧：FF FF 01 04 00 18 05 DD

4.3 WRITE DATA 写数据

示例：ID1转到2048，速度1000

指令帧：FF FF 01 09 03 2A 00 08 00 00 E8 03 D5
应答帧：FF FF 01 02 00 FC

4.4 REG WRITE 异步写

先缓存指令，等待ACTION执行。

4.5 ACTION 执行异步写

指令帧：FF FF FE 02 05 FA（广播无应答）

4.6 SYNC WRITE 同步写

一条指令控制多舵机同时动作。

4.7 SYNC READ 同步读

一次读取多个舵机数据。

4.8 RESET 状态重置

指令帧：FF FF 01 02 0A F2

4.9 位置校准

无参数：校准中位；有参数：校准到指定值。

型号固件支持
SMS≤2.54是
STS≥3.10是
HLS≥3.43是

4.10 参数恢复

FF FF 01 02 06 F6

4.11 参数备份

FF FF 01 02 09 F3

4.12 重启指令

FF FF 01 02 08 F4（无应答）

重启前请先关闭扭矩开关。
